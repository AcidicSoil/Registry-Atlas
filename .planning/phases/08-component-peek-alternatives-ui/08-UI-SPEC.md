# Phase 08 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — keep existing vanilla TypeScript DOM renderers |
| Icon library | none — use text labels/arrows only unless already present |
| Font | Existing system font stack from `public/styles/registry-explorer.css` |

**Contract:** Phase 8 must extend the current dark Registry Atlas design system. Do not add React, shadcn blocks, Radix, icon libraries, iframes, or a new styling framework.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tight text/icon gaps, badge remove affordances |
| sm | 8px | Peek title gap, active filter badge padding, compact row gaps |
| md | 16px | Default card/popover padding, filter bar gaps, related-strip gaps |
| lg | 24px | Section padding, item/detail related-section spacing |
| xl | 32px | Major content/aside gaps |
| 2xl | 48px | Page-level breaks only |
| 3xl | 64px | Not expected in Phase 8 feature UI |

Exceptions: none.

**Density rules:**

- Quick peek popovers must stay compact: target width 260–360px; max width `min(360px, calc(100vw - 32px))`.
- Popover content may use `max-height` and scroll only when unavoidable; the preferred peek is one visual/placeholder plus at most a tiny title.
- Aside/pill-list surfaces must become scrollable when long enough; do not let `.app-inner > main > aside` or `aside > #aside > .pill-list` stretch the whole page.
- Cards must not regain the Phase 7 overcrowding problem. Do not expand all badges inline.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 600 | 1.35 |
| Heading | 18px | 700 | 1.25 |
| Display | 24px | 700 | 1.15 |
| Peek title | 12–13px | 600 | 1.25 |

**Contract:**

- Peek text should be minimal. The component visual is the content.
- Peek may show a tiny name/title above the visual; otherwise avoid descriptive text.
- Filter picker labels should be short and scannable: `Type`, `Category`, `Tag`, `Visual`, `Status`.
- Avoid noisy confidence wording in compact UI. Do not render phrases like `verified item` or `high confidence` as prominent browsing badges.

## Color

Use existing CSS variables from `public/styles/registry-explorer.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg` `#05060a` | Page background |
| Secondary (30%) | `--bg-elevated`, `--bg-soft`, `--bg-softer` | Cards, popovers, sidebars, filter picker, related strip |
| Accent (10%) | `--accent` `#ffd95e` | Primary CTA emphasis, active selected filter, focus outline |
| Secondary accent | `--accent-secondary` `#65d4ff` | Subtle link/related-navigation emphasis |
| Text | `--text-main`, `--text-muted`, `--text-softer` | Body, supporting text, low-priority detail |

Accent reserved for: active filter badges, focused peek trigger outline, primary `View component` / `Open component page` action, selected related item, and essential reset affordances. Do not use accent on every chip or metadata label.

## Interaction Contract

### Quick Peek

| Behavior | Contract |
|----------|----------|
| Trigger | Hover and keyboard focus show the quick peek for route-eligible items. |
| Touch/click | Click/tap opens the Phase 7 item page. It must not be hover-only. |
| Placement | Small anchored popover near the triggering `View component` / peek control. |
| Dismissal | Mouse leave, blur, `Esc`, or outside click/tap closes the peek. |
| Content | Visual first; at most tiny title/name above the visual. |
| No visual | Minimal placeholder: `Preview not available yet` plus `Open component page`. |
| Forbidden | No raw JSON, long metadata, dependency lists, file lists, confidence dumps, or full item-card recreation inside the peek. |

### Filter Picker

| Behavior | Contract |
|----------|----------|
| Entry | A compact `+ Filter` control above results or near the relevant sidebar/content selector. |
| Source | Options are generated from loaded `Registry.itemSummaries`, not hardcoded to a tiny list. |
| Selection | Selecting a value adds a removable active badge/tag. |
| Reset | Always provide clear reset/remove behavior. |
| Scope | Phase 8 must support item type; the picker should allow other metadata dimensions from current data without redesign. |
| Consistency | Discovery, profile, component, and related route/sidebar surfaces should not expose inconsistent filter vocabularies when the same metadata exists. |
| Overflow | Long picker/pill content scrolls. It must not expand cards or sidebars indefinitely. |

### Related / Similar Navigation

| Behavior | Contract |
|----------|----------|
| Placement | Small strip/navigator on item/peek detail surfaces. Do not put alternatives inside every discovery card. |
| Basis | Shared type, category, tags, and available item metadata. |
| Flow | User can click a related component, navigate to it, then continue browsing from that component. |
| Content | Minimal visual/placeholder + component name/registry. |
| Language | Use safe visible labels such as `Similar patterns` or `Related components`. |
| Forbidden | Do not claim `better`, `best`, `correct`, `polished`, or `production-grade` in Phase 8 UI without real evidence. |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Peek trigger | `Peek` or `View component`; use `View component` when only one action fits. |
| Peek placeholder heading | `Preview not available yet` |
| Peek placeholder body | Omit when possible. If needed: `Open the component page to inspect it.` |
| Peek fallback CTA | `Open component page` |
| Filter entry | `+ Filter` |
| Active filter reset | `Reset filters` |
| Active filter remove label | `Remove {filter label}` |
| Related section heading | `Similar patterns` or `Related components` |
| Related section note | `Matched by shared type, category, or tags.` |
| Empty related state | `No similar components in this data set yet.` |
| Empty filter state | `No components match these filters. Reset filters to see all results.` |
| Safety note | `Review third-party registry code before installing.` |

**Forbidden prominent copy:**

- `Verified item`
- `High confidence`
- `Best`
- `Better`
- `Correct way`
- `Production-grade`
- `Approved`
- `Audited`
- `Open raw item route`
- `Raw JSON`
- `Registry JSON`

These phrases may remain only where required by lower-level internal data/tests or non-prominent technical docs, not as normal Phase 8 browsing UI.

## Surface Contracts

### Discovery Results

- Keep rows/cards compact.
- Primary action remains component browsing: `Peek`/`View component`.
- Docs and Registry homepage links are acceptable when they do not form a crowded cluster.
- Do not add full related alternatives, full badge lists, dependencies, files, or raw route/source language to each result row.
- Filter bar/picker may sit above results, not inside every card.

### Registry Profile

- Profile item rows should support the same peek/view behavior as discovery where route-eligible.
- Badge overflow must be solved by removal, compact disclosure, or moving detail to the item page/filter picker — not by rendering every badge inline.
- Remove/demote confidence/status noise from rows.

### Item Detail Page

- The existing component-first page remains the stable destination for click/tap.
- Add related/similar navigator below the main visual/summary area or near the component evaluation area, not above the component itself.
- Preserve copy-only install/view commands.
- Preserve raw JSON exclusion from normal UI.

### Aside / Pill Lists / Component Selectors

- `.app-inner > main > aside` should remain useful at long lengths through scroll constraints.
- `aside > #aside > .pill-list` should be scrollable or grouped when long; do not show only a handful if richer current metadata exists.
- `.app-inner > main > .content` should remain the primary component browsing area and should not be overwhelmed by filter/badge chrome.

### Matrix Boundary

- Phase 8 may improve metadata consistency that will help the matrix later.
- Do not redesign the dynamic matrix, complete all matrix columns, or build the v1.3 interactive matrix in this phase.

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn official | None added in Phase 8 | not required |
| Third-party community registries | Metadata, preview links/images when available, docs/homepage links, copy-only shadcn commands | Escape all rendered strings; safe-link rendering; no code execution; no browser install; no audit/approval claims |

**Safety rules:**

- Treat every registry/item field as untrusted content.
- Use `escapeHtml` for text and `renderExternalLink` for links.
- Do not render remote component code in Atlas.
- Do not iframe/embed third-party pages as the default preview model.
- Do not execute shadcn CLI commands. Copy-only behavior remains required.
- Do not imply Registry Atlas certifies or audits a third-party component.

---

## Accessibility Contract

- Peek triggers must be focusable controls or links.
- Keyboard focus must show equivalent peek information to hover.
- `Esc` must close an open peek.
- Click/tap outside should dismiss transient popovers.
- Filter picker must be keyboard navigable enough for current vanilla DOM patterns.
- Active filter badges need accessible remove labels.
- Related component navigation must be reachable by keyboard and should preserve clear focus after navigation.
- Respect reduced motion: no essential information should depend on animation.

---

## Test Expectations

Planner should include tests for:

- Hover/focus/click/tap state modeling where practical through pure/event-handler tests.
- Peek rendering excludes raw JSON, long metadata, confidence dumps, dependencies, and full card clones.
- Placeholder peek renders minimal copy and `Open component page`.
- Filter options derive from loaded/current item metadata, including item type and expandable category/tag/status/visual dimensions.
- Active filter badges can be added, removed, and reset.
- Long lists/pill areas have classes or markup enabling scroll constraints.
- Prominent `Verified item` / `high confidence` copy is removed or demoted from Phase 8 browsing surfaces.
- Related component derivation is similarity-based and not quality-ranked.
- Related strip/navigator uses safe labels and does not claim better/best/correct.
- Full `pnpm verify` remains the gate.
