# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: Expanded Component Catalog. Generated by `$gsd-ui-phase`, verified against the GSD UI checker dimensions, and consumed by `$gsd-plan-phase`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — manual static CSS in `public/styles/registry-explorer.css` |
| Preset | not applicable |
| Component library | none — vanilla TypeScript DOM renderers |
| Icon library | none — existing UI uses text glyphs and CSS shapes only |
| Font | system-ui, -apple-system, BlinkMacSystemFont, "Chivo", sans-serif |

**Existing visual baseline:** preserve the dark, dense Registry Atlas explorer surface. Phase 5 should extend the current discovery/profile UI rather than introducing a new page shell or external design system.

**Primary visual anchor:** the discovery result row remains the primary object. New catalog-backed items should make the item name, registry namespace, status chip, and command/source actions scannable in one row.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tab shell padding, compact inline offsets |
| sm | 8px | Chip gaps, metadata row gaps, compact button spacing |
| md | 16px | Default section gaps, result row internal spacing |
| lg | 24px | Major panel padding, profile section rhythm |
| xl | 32px | Large card/list group spacing when density allows |
| 2xl | 48px | Major section breaks only |
| 3xl | 64px | Page-level spacing; avoid inside the dense explorer |

Exceptions: existing CSS includes 5px/6px/10px/11px/12px/14px/18px/20px/22px/28px values from earlier shipped UI. Phase 5 should not expand those exceptions; new layout spacing should use the declared 4/8/16/24/32/48/64 scale unless editing an existing selector that already uses a nearby value.

**Touch/click targets:** interactive buttons that add/copy/open catalog actions should keep at least 32px visual height in dense rows and must remain keyboard reachable with visible focus.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px | 600 | 1.3 |
| Body | 14px | 400 | 1.5 |
| Heading | 20px | 600 | 1.25 |
| Display | 28px | 600 | 1.15 |

**Usage rules:**

- Use Label for status chips, metadata chips, button labels, and compact table/list labels.
- Use Body for descriptions, provenance summaries, notes, disabled reasons, and empty/error state body copy.
- Use Heading for profile sections and expanded-catalog panel headings.
- Use Display only for the main route header; do not add new oversized marketing headers.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #05060a | App background, main canvas, dense discovery surface |
| Secondary (30%) | #0b0d15 / #101320 / #151827 | Cards, sidebar, result rows, profile sections, raised surfaces |
| Accent (10%) | #ffd95e | Active tab gradient, primary copy/install CTA, route-eligible/catalog-backed emphasis |
| Secondary accent | #65d4ff | Secondary highlight in gradients, non-primary information emphasis |
| Destructive | #ff6b6b | Destructive or risk-state copy only; not needed for normal Phase 5 actions |

Accent reserved for: active navigation, primary copy/install buttons, catalog-backed verified emphasis, focus rings, and the existing brand logo. Do not use accent for every link or all interactive elements.

**Status colors:** reuse existing `status-*`, `catalog-*`, `confidence-chip`, `partial-data-note`, and muted text patterns. If new catalog states are added, map them to existing status semantics rather than inventing unrelated colors.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Copy install command |
| Secondary CTA | Inspect first |
| Source action | Open raw item route |
| Profile action | View profile |
| Queue action | Add to queue |
| Empty state heading | No catalog-backed components match this search yet |
| Empty state body | Try a component name, registry namespace, category, or tag. Some registries still need manual catalog research. |
| Partial data note | Some components are inferred or unavailable because not every registry exposes a verified item catalog. |
| Error state | Expanded catalog data could not be loaded. Run `pnpm validate:data` and check the generated registry data. |
| Disabled command reason | Command unavailable until Registry Atlas has a valid namespace, item slug, and route-eligible catalog record. |
| Destructive confirmation | None — Phase 5 adds no destructive user action. |

**Tone rules:**

- Use “catalog-backed” for items imported from concrete registry/catalog data.
- Use “inferred” only when the app is matching tags/metadata without a verified item route.
- Use “manual follow-up” for registries that need more research before showing item actions.
- Do not say “safe,” “approved,” “audited,” or “trusted” for third-party registry code.
- Keep command text copy-only: the UI may copy commands and open links, but must not imply the browser runs installs.

## Interaction Contract

| Surface | Required behavior |
|---------|-------------------|
| Discovery result row | Show item label, registry namespace, item slug, category/type, catalog status, confidence/status chip, route/source action, and copy-only commands when route eligible. |
| Registry profile item list | Show the same eligibility and status semantics as discovery results so profile and discovery do not disagree. |
| Install/view commands | Enabled only for validated namespace + slug + route-eligible records. Disabled states must show a concrete reason. |
| Queue controls | Queue only enabled install tokens. Do not encode queue state in the URL. |
| Search empty state | Avoid false “no results” conclusions; mention incomplete catalog coverage when applicable. |
| Keyboard access | Copy, inspect, queue, profile, and raw-route controls must be native buttons/links or equivalent keyboard-reachable controls with visible focus. |
| Feedback | Copy actions should use existing `copy-feedback` role/status behavior or equivalent `aria-live` feedback. |

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn official | Registry directory metadata and CLI command syntax references | not required |
| third-party UI blocks | none — Phase 5 does not install or import third-party UI code into Registry Atlas | not applicable |
| `@delego`, `@delta`, `@diceui` registry metadata/items | Catalog/item metadata only; surfaced as data and copyable commands | copy-only; no browser execution; no code endorsement |

**Safety contract:** Phase 5 may display metadata and copy shadcn `view` / `add` commands for route-eligible items, but it must not install, execute, render, or rehost third-party registry component code.
