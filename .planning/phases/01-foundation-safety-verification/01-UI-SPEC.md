# Phase 1 - UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Phase 1 is a foundation phase. Preserve the existing Registry Atlas explorer surface while making safety, canonical-surface, empty-state, and error-state behavior explicit. Do not introduce a framework migration, shadcn component setup, Tailwind, third-party registry blocks, or a new visual direction in this phase.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | none; keep existing lightweight text glyphs only |
| Font | `system-ui, -apple-system, BlinkMacSystemFont, "Chivo", sans-serif` |

Source notes:
- `components.json`, Tailwind config, and PostCSS config are absent.
- The stack is vanilla TypeScript + Vite, not React/Next/Vite React, so the shadcn initialization gate is not applicable.
- Existing UI styling lives in `public/styles/registry-explorer.css`; Phase 1 should update that stylesheet only when needed for safety, canonical surface cleanup, or state polish.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, table padding accents, compact chip gaps |
| sm | 8px | Compact element spacing, registry card grid gaps |
| md | 16px | Default body padding and form/control grouping |
| lg | 24px | Large panel/header separation |
| xl | 32px | Brand mark size and major layout rhythm |
| 2xl | 48px | Reserved for future page-level vertical breaks |
| 3xl | 64px | Reserved for future full-page section breaks |

Exceptions: Existing non-spacing geometry tokens may remain in Phase 1: `max-width: 1320px`, current border radii, current shadows, and current transition durations. Any new clickable target introduced in Phase 1 must be at least 44px tall/wide on touch viewports or use the existing tab/search/pill target pattern without shrinking it.

## Typography

Use exactly these four role sizes for new or changed Phase 1 UI copy. Existing smaller chip/table sizes may remain unchanged unless the touched element is being redesigned.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 | 1.5 |
| Label | 11px | 400 | 1.4 |
| Heading | 14px | 600 | 1.2 |
| Display | 17px | 600 | 1.2 |

Rules:
- Use only weights `400` and `600` for new or changed UI text.
- Do not add viewport-scaled font sizes.
- New labels should not use negative letter spacing. Existing uppercase tracking may remain, but do not increase it.
- Error detail text must be readable at `11px` minimum and must not be injected as HTML.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#05060a` | App background, primary page shell, main dark surfaces |
| Secondary (30%) | `#0b0d15` | Elevated panels, cards, sidebar, search, tabs, table bodies |
| Accent (10%) | `#ffd95e` | Active tab fill, selected filter pill, metric numbers, focus chips, status dot, empty/error emphasis |
| Destructive | `#ff6b6b` | Destructive actions only |

Accent reserved for: active tab state, selected aside pill state, registry/focus emphasis chips, visible metric numerals, empty-state emphasis, render-error icon or heading, and the local-data status dot.

Additional color rule: Existing `#65d4ff` may remain as a support color for external links and component names. It is not the primary accent and must not be used for safety, approval, or endorsement states.

Destructive color is reserved for actual deletion/removal actions. Phase 1 should avoid destructive in-app actions; repository artifact deletion is an implementation concern, not a browser UI control.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Verify foundation |
| Empty state heading | No registries match this filter. |
| Empty state body | Clear the search or choose a different group. |
| Error state | Something went wrong while rendering this view. Try clearing the search or reloading the page. |
| Destructive confirmation | Remove legacy deployed artifact: Confirm the file is outside the canonical Registry Atlas app surface before deleting it. |

Copy rules:
- Keep user-facing copy neutral and precise. Do not imply Registry Atlas audits, approves, verifies, or endorses community registry code.
- For rejected unsafe links, show non-alarming copy: `Link unavailable: unsupported URL protocol.`
- For render failures, do not expose raw exception details in visible UI. Log details with the existing `Registry Explorer:` prefix and show the safe error copy above.
- Keep the existing search placeholder unless touched: `Filter by name, focus, or tag`.
- Keep the current brand label `Registry Atlas`; Phase 1 may update the document title from `Component Registry Explorer` to `Registry Atlas` as part of canonical-surface cleanup.

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn official | none | not required |
| third-party registries | none | not applicable - no third-party UI registry blocks approved for Phase 1 |

Safety rules for Phase 1:
- Do not add shadcn registry blocks or third-party UI packages.
- Do not execute installs from the browser.
- All imported registry text must pass through one shared HTML escaping/text helper or DOM `textContent`.
- If string rendering remains, escape `&`, `<`, `>`, `"`, and `'` before interpolation.
- All rendered links must be parsed with `new URL()` and allow `https:` only by default.
- External anchors must keep `target="_blank"` and `rel="noreferrer"`.
- Invalid, missing, or unsupported links render as unavailable text, not as clickable anchors.

## Interaction Contract

## Visual Hierarchy

Primary focal point is the search/filter control and current result count. Secondary focus is the active tab and selected aside pill. Registry cards, rows, and matrix entries remain the main scanning surface.

| Surface | Contract |
|---------|----------|
| Canonical app surface | `index.html`, `public/styles/registry-explorer.css`, and `src/registry-explorer/entry.ts` remain the only browser app entry surface. Legacy/starter deployable artifacts must not be reachable from the shipped app. |
| Search | Search stays client-side and immediate. Empty results must preserve layout and show the empty-state copy above. |
| Tabs | Tabs keep `role="tablist"`/`role="tab"` semantics. Active state must be visible through color and class state. Invalid `data-view` values must be ignored instead of entering app state. |
| Aside filters | Focus/component pills remain button controls. Active pill stays visually distinct and does not resize the layout. |
| External links | Valid links open in a new tab with `rel="noreferrer"`. Invalid links must not render as anchors. |
| Render error fallback | Error fallback must preserve the content panel layout and use safe text insertion. Exception details stay in `console.error`, not visible `innerHTML`. |
| Responsive layout | Preserve the existing two-column desktop layout, one-column layout below 900px, stacked header below 720px, and compact shell below 520px. |
