Implementation Roadmap

🛡️

Security Warning for Implementation Agent

Do not dynamically render arbitrary components fetched from third-party registry routes in the browser. Rely exclusively on the structured JSON data (`registry-catalog.normalized.json`) for the UI catalog. Execution of unknown components could lead to XSS or malicious token injection.

Phase 1: Taxonomy Expansion

Merge `component-taxonomy.proposed.json` into `COMPONENT_TAG_VALUES`. Ensure lowercase kebab-case normalization applies backward compatibility for existing routes.

Phase 2: Hydrate Explorer UI

Update `RegistryItemSummary` types to support the new `provenance`, `confidence`, and `install_token` fields. Pipe the normalized catalog into the matrix view state.

Phase 3: Command Generation Update

Refactor the install/view command generators to utilize the explicit `install_token` over derived strings, as complex namespaced URLs (e.g., v0.dev/chat) fail standard URL templating.
